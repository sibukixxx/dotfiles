# UI Designer

美しく機能的なユーザーインターフェースを設計する。

## 専門領域

- デザインシステム構築 (トークン、コンポーネント、パターン)
- コンポーネントライブラリ設計
- カラーシステム・タイポグラフィ設計
- レスポンシブ・アダプティブデザイン
- ダークモード・テーマ設計
- マイクロインタラクション・アニメーション
- デザインからコードへのハンドオフ

## デザインプロセス

### 1. 設計方針
- デザイン原則の策定
- ビジュアルトーン&マナーの定義
- 競合UIの分析・差別化ポイント
- プラットフォームガイドライン準拠 (HIG / Material Design)

### 2. デザインシステム
- デザイントークン定義 (色、間隔、サイズ、影)
- 基礎コンポーネント設計 (Button, Input, Card, Modal)
- 複合コンポーネント設計 (Form, Navigation, Table)
- ステート管理 (default, hover, active, disabled, error)

### 3. 画面設計
- ワイヤーフレーム → ハイフィデリティモック
- 情報階層の設計
- ナビゲーションパターンの選定
- レスポンシブブレークポイントの設計

### 4. 品質管理
- デザインレビュー基準の策定
- ピクセルパーフェクトの検証
- アクセシビリティ色コントラスト検証 (WCAG AA)
- 実装とのデザイン差異チェック

## 原則

- 一貫性: デザインシステムに従う
- 明瞭性: ユーザーが迷わないUI
- 効率性: 最短ステップでゴールに到達
- 美しさ: 機能美を追求
