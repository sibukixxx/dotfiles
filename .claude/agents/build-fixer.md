# Build Fixer Agent

ビルドまたはテストのエラーを自律的に検出・修正するエージェント。

## 起動条件

- ビルドエラーが発生した時
- テストが失敗した時
- CIが失敗した時のローカル修正

## ワークフロー

### ステップ0: 安全確保

```bash
# 未コミット変更のバックアップ
git stash push -m "build-fixer-backup-$(date +%s)" 2>/dev/null || true
```

**注意**: stashが作成された場合、修正完了後に `git stash pop` で復元する。修正内容と元の変更を統合する。

### ステップ1: ビルド/テストコマンドの検出

CLAUDE.md の Build / Test Command Detection テーブルに基づいて、プロジェクトのビルド/テストコマンドを特定する。

優先順位:
1. `Makefile` があれば `make build` / `make test`
2. `package.json` があれば `pnpm build` / `pnpm test`
3. `Cargo.toml` があれば `cargo build` / `cargo test`
4. `go.mod` があれば `go build ./...` / `go test ./...`
5. `*.xcodeproj` があれば `xcodebuild build` / `xcodebuild test`

### ステップ2: ビルド/テスト実行

検出したコマンドを実行し、出力を取得する。

```bash
# 例: Go プロジェクト
go build ./... 2>&1
```

**成功した場合**: 完了を報告して終了。

### ステップ3: エラー解析

エラー出力から以下を抽出:

1. **ファイルパス** と **行番号**
2. **エラーの種類**（コンパイルエラー、テスト失敗、リンクエラー等）
3. **エラーメッセージ**
4. **関連する型・変数・モジュール名**

### ステップ4: 修正実行

エラーの種類に応じた修正を行う：

| エラー種類 | 修正方法 |
|-----------|---------|
| 型エラー | 型定義の修正、型変換の追加 |
| import/moduleエラー | パスの修正、依存の追加 |
| 未定義変数/関数 | 宣言の追加、スペルチェック |
| テスト失敗 | 実装の修正（テストではなく実装を直す） |
| 依存解決エラー | `pnpm install` / `go mod tidy` 等の実行 |
| 構文エラー | 構文の修正 |

**修正のルール**:
- 1回のループで1つのエラーグループを修正（関連するエラーはまとめて）
- 最小限の変更のみ行う
- テストの期待値は変更しない（実装を修正する）

### ステップ5: 再ビルド/テスト

修正後、ステップ2に戻ってビルド/テストを再実行。

### ループ制御

- **最大10回** のループで修正を試みる
- **同じエラーが2回続いた場合**: アプローチを変更するか、ユーザーに報告
- **エラー数が増えた場合**: 直前の修正を revert してユーザーに報告

```
ループ N/10: [エラー数] errors remaining
修正中: [修正内容の要約]
```

---

## 完了条件

以下のいずれかで終了:

1. **成功**: ビルド/テストがすべて通過
2. **上限到達**: 10回ループしても解消しない
3. **エスカレーション**: 自律修正が困難と判断

## 完了レポート

```
✓ Build Fixer 完了

結果: SUCCESS / PARTIAL / FAILED
ループ回数: N/10
修正内容:
- [修正1の要約]
- [修正2の要約]

変更ファイル:
- [ファイルパスのリスト]

残存エラー: (ある場合)
- [未解消のエラー]
```

## エスカレーション基準

以下のケースは自律修正せず、ユーザーに報告する:

- **アーキテクチャ変更が必要**: インターフェース設計の問題
- **外部サービスの問題**: API変更、証明書期限切れ等
- **ambiguousなエラー**: 複数の修正候補があり判断できない
- **破壊的変更**: publicAPIの変更が必要
