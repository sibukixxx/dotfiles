# follow-up: フォローアップメール自動生成スキル

商談・MTG後のフォローアップメールを議事録やメモから自動生成する。

## 前提条件

### 必須
- qmd がインストール済み（Obsidian ナレッジベース検索）

### オプション（MCP連携で強化）
- Circleback MCP → AI議事録から自動取得
- Gmail MCP → 下書きに直接保存
- Google Calendar MCP → MTG情報の自動取得

## 使い方

```
/follow-up [会社名 or MTGの説明]
```

引数なしの場合、直近のMTGメモやノートから推測する。

## Workflow

### Step 1: MTG情報の収集

以下の優先順で情報を取得する:

1. **ユーザー指定のファイル**: 引数でファイルパスが渡された場合はそれを読む
2. **qmd 検索**: `qmd query "[会社名] MTG"` で直近のノートを検索
3. **Circleback MCP**（利用可能な場合）: 直近のMTG議事録を取得
4. **対話ヒアリング**: 情報が不足する場合はユーザーに確認
   - 相手先（会社名・担当者名）
   - MTGで話した主なトピック
   - 決まったこと・宿題

### Step 2: 要点抽出

議事録・メモから以下を構造化する:

- **決定事項**: MTGで合意したこと
- **アクションアイテム**: 誰が・何を・いつまでに
- **次回MTGの予定**（あれば）
- **相手が特に関心を示したポイント**

### Step 3: メール生成

以下のガイドラインでフォローアップメールを作成:

- **テンプレ感を出さない**: 実際にMTGで話した具体的な内容に触れる
- **簡潔**: 3-5段落以内
- **構造**: お礼 → 要点振り返り → アクションアイテム → 次回予定
- **トーン**: ビジネスだが堅すぎない、相手との関係性に合わせる

### Step 4: 出力

メール本文をマークダウンファイルとして保存し、ターミナルにも表示する。

## Output Format


## Save Location

`notes/docs/records/follow-up/[会社名]-YYYY-MM-DD.md`

## 安全ガードレール

- **メールは絶対に自動送信しない**: 必ず下書きとして保存・表示のみ
- ユーザーが内容を確認してから手動で送信する
- Gmail MCP 連携時も下書き保存のみ（send権限は付与しない）

## MCP 連携なしでの動作

MCP なしでも以下のフローで動作する:

1. ユーザーがMTGの要点を口頭で伝える（音声入力OK）
2. qmd で過去の関連ノートを補完
3. メール下書きをローカルファイルに出力
4. ユーザーがコピペでメール送信

## Quality Checklist

- [ ] MTGの具体的な内容に触れているか（テンプレ感がないか）
- [ ] アクションアイテムに担当と期限が明記されているか
- [ ] 相手との関係性に合ったトーンか
- [ ] 次回予定が含まれているか（あれば）
- [ ] 送信ではなく下書き保存になっているか
