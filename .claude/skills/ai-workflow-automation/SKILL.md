# AI Workflow Automation

業務オペレーションを Claude（または同等 LLM）と orchestration ツール（n8n / Zapier / Make / 自前コード）で自動化する。**ツール起点ではなくワークフロー起点** で設計する。

## 発動条件

- 「ツールを雇う前に AI で自動化したい」「運用に時間取られすぎ」
- メール仕分け / レポート生成 / コンテンツパイプライン / 請求処理 / リード qualification の自動化
- n8n / Zapier / Make を使ってエージェント連携を設計する相談
- 既存の手作業ワークフローを AI 化したい

## 重要な前提（最初に伝える）

ネット記事の自動化レシピは **3つの致命的な欠落** を持つことが多い。本スキルは以下を必須とする。

1. **観測性（o11y）が初手**。トレースが無いワークフローは改善不能。`.claude/CLAUDE.md` の AI Agent Observability ルールに従う。
2. **テスト駆動**。ワークフローはコード。Red → Green → Refactor で設計する。
3. **セキュリティはオプションではない**。PII 取扱い・シークレット管理・prompt injection 対策を最初の workflow から組み込む。

## 推奨ビルド順（よくある記事は順番が逆）

| 順 | ワークフロー | 理由 |
|---|------------|------|
| 1 | **Report Factory** | 低リスク。スケジュールトリガ・構造化データ・内部向け。失敗しても顧客に飛び火しない |
| 2 | **Content Engine** | 中リスク。最終出力に人間レビューが入る前提なら安全 |
| 3 | **Email Operations Center** | 高リスク。顧客対面・分類曖昧。1, 2 で仕組みを成熟させてから着手 |

ネット記事は逆順（Email Center 最初）を推すが、**新規構築の最初に顧客対面のものを置くな**。誤分類で取引先に変な返信が飛ぶ事故が起きる。

## 5フェーズ実装

詳細は [references/5-phase-implementation.md](references/5-phase-implementation.md) を参照。各フェーズで本リポジトリの規約を統合：

| Phase | 内容 | この repo で必須にすること |
|-------|------|------|
| 1 | ワークフロー棚卸し | 🟢🟡🔴 分類 + **データ機密性レベル**（公開/社内/PII）も付与 |
| 2 | アーキテクチャ設計 | 5コンポーネント + **トレース設計** + **コスト想定** + **失敗時挙動** |
| 3 | コア3ワークフロー構築 | TDD（テストを先に書く）。Report → Content → Email の順 |
| 4 | ワークフロー連携 | 疎結合・イベントバス。**ベンダーロックイン回避**（プロンプトを抽象化） |
| 5 | 監視・改善 | success rate / latency / quality に加え **cost per run** と **トレース** |

## 出力フォーマット

ユーザーから相談を受けたら：

1. **業務全体マップ**を1行で要約（例: 「BtoB SaaS の営業 → サポート → 経理」）
2. **🟢🟡🔴 + データ機密** で棚卸しさせる（テンプレ提供）
3. **最初の1ワークフロー**を Report Factory から提案
4. **5コンポーネント設計図**（trigger / input / AI / output / QC）を作る
5. **o11y・テスト・セキュリティのチェックリスト**を必ず添付
6. 期待する **コスト・処理時間・成功率** を数値で書く（仮置き OK、後で計測上書き）
7. **ロールバック手順**を必ず書く

## 決定木：何から手を付けるか

```
ユーザーの主訴
├── 「全部自動化したい」(漠然)
│   → Phase 1（棚卸し）から。1セッションで全部はやらない
├── 「メールの仕分けを自動化したい」
│   → 警告: 顧客対面で最初に置くなら人間レビュー強制
│   → 先に Report Factory を作って仕組み習熟を勧める
├── 「週次レポートを自動化したい」
│   → ✅ ベスト。Report Factory テンプレートから
├── 「コンテンツパイプライン」
│   → Content Engine テンプレートから
├── 「既に動いている自動化を改善したい」
│   → トレース有無を最初に確認。無ければ o11y 入れる方が先
└── 「n8n / Zapier どれ？」
    → references/orchestrator-selection.md を参照
```

## やってはいけないこと

- **顧客対面ワークフローを最初に作る**（誤動作の事故が学習コスト以上に痛い）
- **トレース無しで本番投入**（改善ループが回らない）
- **「AI が書いた」を信じてレビュー無しで送信**
- **プロンプトを Git 管理しない**（バージョン無しで品質劣化が追跡できない）
- **コスト上限を設定しない**（暴走でクレカ請求が爆発する事故）
- **PII を平文でログに残す**（GDPR / 個人情報保護法違反）
- **モデルロックイン**（Claude 専用 prompt しか書かない → 移行不能）
