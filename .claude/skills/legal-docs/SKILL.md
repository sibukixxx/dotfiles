# legal-docs: 法務ドキュメント作成・レビュー

契約書・法務文書の作成とレビューを支援する。日本法準拠。

> **免責**: 本スキルはドラフト作成の支援ツールであり、法的助言ではない。
> 重要な契約は必ず弁護士の確認を受けること。

## 対応ドキュメント

| 種別 | コマンド例 | 詳細 |
|------|-----------|------|
| NDA（秘密保持契約） | `/legal-docs NDA` | 双方向/片方向 |
| 業務委託契約 | `/legal-docs 業務委託` | フリーランス/法人間 |
| 利用規約 | `/legal-docs 利用規約` | SaaS/Webサービス |
| プライバシーポリシー | `/legal-docs PP` | 改正個人情報保護法準拠 |
| 雇用契約 | `/legal-docs 雇用契約` | 正社員/アルバイト/業務委託 |
| 定款 | `/legal-docs 定款` | 株式会社/合同会社 |
| 督促状 | `/legal-docs 督促状` | 未払い催告 |
| 契約レビュー | `/legal-docs レビュー` | リスク抽出+修正案 |
| 英文契約レビュー | `/legal-docs 英文レビュー` | 日本語要約+落とし穴指摘 |

## Workflow

### Step 1: ドキュメント種別の特定

ユーザーの要望から作成すべき文書を特定する。不明な場合はヒアリング。

### Step 2: 必要情報のヒアリング

各テンプレートに必要なパラメータを確認（詳細は `references/contract-templates.md`）:
- **当事者**: 会社名・個人名
- **目的**: 契約の対象（プロジェクト、業務内容等）
- **期間**: 有効期間・自動更新の有無
- **準拠法・管轄**: デフォルトは日本法・東京地裁
- **固有条件**: 報酬、知財、免責等

### Step 3: ドラフト作成

テンプレートとパラメータに基づきドラフトを生成。以下を遵守:
- **日本の法律に準拠**: 会社法、労働基準法、消費者契約法、個人情報保護法等
- **自社利益の保護**: リスクを最小化する条項を含める
- **明確な日本語**: 法的に曖昧な表現を避ける
- **商習慣に適合**: 日本のビジネス慣行に沿った構成

### Step 4: レビュー（契約レビューの場合）

レビュー時のチェックポイントは `references/review-checklist.md` を参照。

### Step 5: 出力

マークダウン形式で出力。ユーザーが求めればファイルに保存。

## 作成ルール

- 条項番号を連番で付与（第1条、第2条...）
- 各条項にタイトルを付ける
- 定義条項を冒頭に配置
- 一般条項（準拠法、管轄、協議事項）を末尾に配置
- 空欄パラメータは `[____]` で明示し、ユーザーに埋めてもらう

## Troubleshooting

### どのテンプレートを使うべきかわからない場合
ユーザーに「何のために」「誰と」「何を」契約するかを確認。最適なテンプレートを提案。

### 特殊な条項が必要な場合
標準テンプレートに追加条項として挿入。「本契約に定めのない事項」の協議条項で補完。

### 海外取引で準拠法が異なる場合
日本法以外の準拠法は専門家の確認を強く推奨。英文契約のレビュー・要約は対応可能。
