# /retro — エンジニアリング振り返り

コミット履歴、作業パターン、コード品質メトリクスを分析する包括的な振り返りレポートを生成する。

## 使い方

- `/retro` — デフォルト: 過去7日間
- `/retro 24h` — 過去24時間
- `/retro 14d` — 過去14日間
- `/retro 30d` — 過去30日間
- `/retro compare` — 今期 vs 前期の比較
- `/retro compare 14d` — 明示的なウィンドウで比較

## Instructions

引数 `$ARGUMENTS` をパースして期間を決定する。デフォルトは7日間。

### ワークフロー概要

1. **デフォルトブランチ検出** — `gh repo view` で取得
2. **データ収集** — 7つの git コマンドを並列実行（詳細は `references/git-commands.md`）
3. **メトリクス算出** — コミット数、LOC、テスト比率、セッション数等
4. **時間分布分析** — 時間別ヒストグラム、ピーク時間、深夜クラスタ
5. **セッション検出** — 45分ギャップでセッション区切り（ディープ/ミディアム/マイクロ）
6. **コミットタイプ内訳** — Conventional Commit プレフィックスで分類
7. **ホットスポット分析** — 変更頻度上位10ファイル
8. **PR サイズ分布** — Small/Medium/Large/XL
9. **フォーカススコア** — トップレベルディレクトリへの集中率
10. **チームメンバー分析** — 個人ディープダイブ + 称賛 + 成長機会
11. **ストリーク追跡** — 連続コミット日数
12. **履歴比較** — 前回の振り返りとのデルタ算出
13. **履歴保存** — `.context/retros/` にJSON保存
14. **ナラティブ出力** — ツイート可能サマリー + 構造化レポート

出力フォーマットの詳細は `references/output-format.md` を参照。

## Compare モード

`/retro compare` 実行時:
1. 今期のメトリクスを算出
2. 前の同期間のメトリクスを `--since` と `--until` で算出
3. サイドバイサイド比較テーブルとデルタ・矢印を表示
4. 最大の改善と後退をハイライトするナラティブ

## トーン

- 励ましつつ率直。甘やかさない
- 具体的 — 常に実際のコミット/コードに基づく
- 一般的な称賛は省略 — 何が良かったか・なぜ良かったかを具体的に
- 改善はレベルアップとしてフレーミング、批判ではない
- 合計 2000-4000 words
- データはmarkdownテーブルとコードブロック、解釈はプロース
- 出力は会話に直接 — .context/retros/ JSON のみファイル書き出し

## Troubleshooting

### git fetch が失敗する場合
リモートが設定されているか `git remote -v` で確認。fork リポジトリの場合は upstream を指定。

### コミットが検出されない場合
`--since` の期間が短すぎないか確認。ブランチが正しいか `git branch -a` で確認。

### 前回の振り返りが見つからない場合
初回実行では比較をスキップする。`.context/retros/` ディレクトリの存在を確認。
