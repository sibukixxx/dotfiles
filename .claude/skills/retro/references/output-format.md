## 分析ルール

### 時間分布ヒストグラム

```
Hour  Commits  ████████████████
 09:    8      ████████
 14:    5      █████
```

ピーク時間、デッドゾーン、深夜コーディングクラスタを特定。

### 作業セッション検出

連続コミット間の **45分ギャップ** でセッションを区切る:
- **ディープセッション** (50分以上)
- **ミディアムセッション** (20-50分)
- **マイクロセッション** (20分未満)

### コミットタイプ内訳

```
feat:     20  (40%)  ████████████████████
fix:      27  (54%)  ███████████████████████████
refactor:  2  ( 4%)  ██
```

fix比率が50%超 → フラグ（「ship fast, fix fast」パターン）。

### ホットスポット分析

変更頻度上位10ファイル。以下をフラグ:
- 5回以上変更されたファイル（チャーンホットスポット）
- テストファイル vs 本番ファイルの割合

### PR サイズ分布

- **Small** (<100 LOC)
- **Medium** (100-500 LOC)
- **Large** (500-1500 LOC)
- **XL** (1500+ LOC) — ファイル数とともにフラグ

### フォーカススコア

コミットの最多トップレベルディレクトリへの集中率。高い = 集中的な深い作業。

### チームメンバー分析

各コントリビューター（自分含む）について:
1. コミット数・LOC
2. フォーカス領域（上位3ディレクトリ）
3. コミットタイプミックス
4. セッションパターン
5. テスト規律（テストLOC比率）
6. 最大Ship

**自分（"You"）**: 最も詳細に分析。
**各チームメート**: 2-3文の作業要約 + **称賛**（1-2個、コミットに基づく具体的なもの）+ **成長機会**（1個、建設的な提案）。
