# サブスクリプション会計（『サブスク会計学』ベース）

## サブスクリプションビジネスの会計特性

### 従来型販売との根本的な違い

| 項目 | 従来型（売切り） | サブスクリプション |
|------|-----------------|-------------------|
| 収益認識 | 引渡し時に一括 | 契約期間にわたり按分 |
| B/S特徴 | 売掛金中心 | 契約負債（前受収益）が大 |
| P/L特徴 | 売上と原価が対応 | 初期は赤字構造（CAC先行） |
| CF特徴 | 売上≒入金 | 入金先行、収益認識は後追い |
| KPI | 売上高、粗利率 | MRR、Churn、NRR、LTV/CAC |

### SaaS企業のP/L構造

```
MRR × 12 = ARR（年間経常収益）

【収益構成】
新規MRR（New MRR）
+ 拡張MRR（Expansion MRR）：アップセル・クロスセル
- 縮小MRR（Contraction MRR）：ダウングレード
- 解約MRR（Churned MRR）
= 純増MRR（Net New MRR）

【費用構造（SaaSの典型）】
売上原価: ホスティング費、カスタマーサクセス人件費、サードパーティ費
  → SaaSグロスマージン 70-85%が健全
S&M（Sales & Marketing）: CAC（顧客獲得費用）の大部分
R&D（Research & Development）: 開発費
G&A（General & Administrative）: 管理費
```

## SaaS KPI の会計的意味

### MRR/ARR（月次/年次経常収益）

```
MRR = Σ（各顧客の月額課金額）

ARR = MRR × 12

【MRR分解】
期末MRR = 期首MRR + New MRR + Expansion MRR - Contraction MRR - Churned MRR
```

**会計上の注意**:
- MRR/ARRは「契約ベース」で計算（GAAP収益とは一致しない場合あり）
- 初期設定料やプロフェッショナルサービスは通常MRRに含めない
- 年間一括払いの場合: 入金額≠MRR（月割りで算入）

### チャーン（解約率）

```
【収益ベースチャーン（Revenue Churn）】※ロゴチャーンより重要
月次収益チャーン = 当月の解約・縮小MRR ÷ 前月末MRR

【ネットレベニューリテンション（NRR）】
NRR = (期首MRR + Expansion - Contraction - Churned) ÷ 期首MRR × 100%

目標: NRR > 100%（既存顧客だけで成長）
  優秀: 120%+
  健全: 100-120%
  要改善: <100%

【ロゴチャーン（顧客数ベース）】
月次ロゴチャーン = 当月解約顧客数 ÷ 前月末顧客数
```

### LTV / CAC（ユニットエコノミクス）

```
LTV（顧客生涯価値） = ARPA × 粗利率 ÷ 月次チャーン率
  ARPA = MRR ÷ 顧客数（顧客あたり平均月額収益）

CAC（顧客獲得費用） = (S&M費用) ÷ 新規顧客数

LTV/CAC比率:
  目標: > 3x
  3x未満: 獲得効率が悪い / チャーンが高い
  5x超: 成長投資の余地がある可能性

CAC回収期間 = CAC ÷ (ARPA × 粗利率)
  目標: 12ヶ月以内
```

### Rule of 40

```
Rule of 40 = 売上成長率(%) + 営業利益率(%) ≥ 40%

例:
  成長率 50% + 利益率 -10% = 40% → 合格
  成長率 20% + 利益率 20% = 40% → 合格
  成長率 15% + 利益率 15% = 30% → 未達
```

SaaS企業の成長と収益性のバランス指標。上場SaaS企業の評価にも使用。

## サブスクリプションの収益認識

### 契約負債（前受収益）の処理

```
【年間契約・一括払いの例: ¥1,200,000/年】

契約開始時（4月1日）:
  現金預金 1,200,000 / 契約負債 1,200,000

毎月末（4月30日〜翌3月31日）:
  契約負債 100,000 / 売上高 100,000
```

### 複合契約の分解

SaaSでよくある複合契約の履行義務識別:

| 構成要素 | 別個の履行義務? | 認識パターン |
|---------|---------------|-------------|
| SaaSライセンス | ○ | 契約期間にわたり均等 |
| 初期設定・導入支援 | 要判定（※） | 一時点 or 期間按分 |
| カスタマイズ開発 | ○（別個なら） | 進捗度に応じて |
| トレーニング | ○ | 提供時に一時点 |
| データ移行 | 要判定 | 一時点 |
| プレミアムサポート | ○ | サービス期間にわたり |

※初期設定が SaaS利用に不可欠で、それ自体に独立した価値がない場合 → SaaSライセンスと一体の履行義務

### 変動対価

| パターン | 処理 |
|---------|------|
| ボリュームディスカウント | 期待値法で見積り |
| 利用量ベース課金 | 利用実績に基づき認識 |
| 成功報酬型 | 最頻値法（変動の不確実性が解消するまで制限） |
| 解約ペナルティ免除 | 免除見込みを変動対価として見積り |

### 契約獲得コストの資産計上（IFRS 15.91-94）

```
資産計上の要件:
  ✓ 契約獲得に直接起因するコスト（営業コミッション等）
  ✓ 回収可能
  ✓ 契約がなければ発生しなかった増分コスト

償却: 財・サービスの移転パターンに従って（通常は契約期間 or 顧客寿命）

実務上の簡便法:
  契約獲得コストの償却期間が1年以内 → 発生時に費用処理可
```

**SaaSの例**: 営業インセンティブ（年間契約のコミッション）を契約期間にわたり償却

## SaaS企業の財務モデリング

### コホート分析

```
【月次コホート収益推移】
                 月1    月2    月3    月4    月5    月6
2025年1月組   1,000    980    970    960    960    980
2025年2月組   1,200  1,180  1,170  1,160  1,160
2025年3月組   1,500  1,470  1,460  1,450
→ 各コホートの収益維持率（リテンションカーブ）を分析
→ 初期の急減後に安定するか（スマイルカーブ）を確認
```

### SaaS財務計画の構造

```
1. トップライン（ARRモデル）
   期末ARR = 期首ARR × NRR + New ARR

2. 費用構造（ARR比率で計画）
   COGS: 15-30% of ARR
   S&M:  30-50% of ARR（成長フェーズ）
   R&D:  15-25% of ARR
   G&A:  10-15% of ARR

3. キャッシュフロー
   営業CF ≈ 当期純利益 + 契約負債増加 + SBC + 減価償却
   （前受収益の増加がCFを改善するのがSaaSの特徴）
```

### SaaS評価指標

| 指標 | 計算 | 目安 |
|------|------|------|
| EV/ARR | 企業価値 ÷ ARR | 成長率により5x-30x |
| EV/Revenue | 企業価値 ÷ 売上高 | SaaSは高めに出る |
| Magic Number | Net New ARR ÷ 前Q S&M費用 | >0.75で健全 |
| Burn Multiple | Net Burn ÷ Net New ARR | <2xが目標 |
| CAC Payback | CAC ÷ (ARPA × GM%) | <18ヶ月 |

## 日本のSaaS企業の開示実務

### 有報でのKPI開示

上場SaaS企業が任意開示する主要KPI:
- ARR / MRR
- 解約率（チャーンレート）
- ARPU / ARPA
- NRR（開示する企業が増加中）
- 顧客数・有料顧客数
- Rule of 40

### IFRSとSaaS

日本のSaaS上場企業のIFRS適用が増加:
- のれん非償却 → M&Aに有利
- 開発費の資産計上 → P/L改善
- IFRS 15はJ-GAAPとほぼ同じ（収斂済み）
- IFRS 16のリースオンバランスは影響限定的（SaaSはアセットライト）
