# コンテンツマーケ運用プレイブック

## 1. 基本原則

- 1コンテンツ1目的にする
- チャネルごとに最適化するが、メッセージの芯は揃える
- 認知施策でもCV導線を消さない
- 制作だけで終わらせず、配信と再利用まで設計する

## 2. ファネル別の役割

| 段階 | 目的 | 代表コンテンツ |
|------|------|---------------|
| TOFU | 認知獲得 | SNS、ショート動画、業界解説記事 |
| MOFU | 比較検討 | 事例、ウェビナー、比較記事、ホワイトペーパー |
| BOFU | 商談化 | 導入事例、料金FAQ、提案資料、デモ導線 |
| Retention | 継続・紹介 | 活用ガイド、アップデート告知、顧客事例 |

## 3. メッセージ設計

### 推奨テンプレート

1. 誰に向けた話か
2. 何が変わるのか
3. なぜ今なのか
4. 何を根拠に言えるのか
5. 次に何をしてほしいか

### 証拠の種類

- 数字
- 顧客事例
- 実演
- 第三者評価

## 4. SEOの実務基準

- 検索意図を `Know / Do / Compare / Buy` に分ける
- 主要キーワードと関連語を事前定義する
- タイトル、H2、導入文、FAQに検索意図を反映する
- 既存記事とのカニバリを避ける
- 公開後も順位・CTR・CVを見て更新する

## 5. SNS運用の基本

### X

- 冒頭1行で止める
- 1投稿1論点
- 画像・図解・短尺動画の再利用を前提にする

### LinkedIn

- 専門性と一次情報を重視する
- 個人の学びと会社の見解を自然に接続する
- 採用広報では仕事内容、裁量、成長環境を具体化する

## 6. コンテンツ評価指標

### 認知

- インプレッション
- リーチ
- 指名検索

### 関心

- 保存率
- クリック率
- 滞在時間

### 商談化

- CVR
- MQL数
- SQL化率
- パイプライン寄与額

## 7. 運用ループ

1. 仮説を置く
2. コンテンツを制作する
3. チャネル別に配信する
4. 指標を見る
5. フック、見出し、CTA、配信時間を改善する

## 8. アンチパターン

- なんとなく毎日投稿する
- 成果指標をエンゲージメントだけで見る
- 採用広報と顧客向け広報を同じ声色で書く
- 記事制作に時間をかけすぎて配信が弱い
