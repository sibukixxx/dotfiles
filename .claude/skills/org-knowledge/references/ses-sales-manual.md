# SES営業マニュアル

## SES営業の基本フロー

```
1. 案件情報収集 → 2. マッチング → 3. 提案 → 4. 面談 → 5. 契約 → 6. 稼働開始 → 7. フォロー
```

## 1. 案件情報収集

### 情報源
- 協力会社からの案件票
- 求人サイト（Freelance, レバテック等）
- 既存クライアントからの追加依頼
- 業界交流会・イベント

### 案件情報の整理項目
- 案件名・概要
- 必要スキル（MUST/WANT）
- 単価レンジ（上限/下限）
- 期間（開始日〜、長期/短期）
- 勤務場所（常駐/リモート/ハイブリッド）
- 面談回数・選考フロー

## 2. マッチング

### スキルシートの要件
- 職務経歴（直近3〜5案件を詳細に）
- 技術スキル一覧（経験年数付き）
- 資格
- 自己PR（案件に合わせてカスタマイズ）

### マッチング判定基準
- MUST要件の充足率: 80%以上
- 経験年数: 案件要件の±1年以内
- 案件の商流: 2次請けまでを推奨

## 3. 単価交渉

### 単価の構成
```
クライアント支払い単価 = エンジニア報酬 + 粗利（25%目標）
```

### 交渉のポイント
- 市場相場を事前調査（言語・フレームワーク・経験年数別）
- 初回は若干高めに提示（5-10%上乗せ）
- 長期契約の場合は単価据え置きを交渉材料に
- スキルアップによる単価改定のタイミングを合意

## 4. 稼働後フォロー

### 定期確認（月1回）
- エンジニア側: 業務内容、人間関係、スキルアップ
- クライアント側: パフォーマンス、コミュニケーション、継続意向

### 契約更新
- 更新3ヶ月前に継続意向を確認
- 更新時に単価改定の検討
- 終了の場合は次案件の準備を並行開始
