# ブランドガイドライン

## トーン&マナー

### ブランドボイス
- **プロフェッショナル**: 専門性を感じさせるが、堅すぎない
- **フレンドリー**: 親しみやすいが、馴れ馴れしくない
- **データドリブン**: 主張には数字や根拠を添える
- **チャレンジング**: 現状維持ではなく、変革を志向する

### OK / NG 表現

| NG | OK |
|----|-----|
| 弊社 | 当社（対外文書）/ 私たち（カジュアル文脈） |
| お客様各位 | [会社名]様 / ご担当者様 |
| させていただく（多用） | いたします / します |
| 〜的な | 〜のような / 〜に近い |

## 表記統一ルール

### 英数字
- 半角を使用: `AI`, `SaaS`, `100万円`
- カタカナ語の後の英語は半角スペースで区切る: `エンジニア SES`

### 固有名詞
- 正式名称を使用（初出時）: `Amazon Web Services (AWS)`
- 2回目以降は略称OK: `AWS`

### 技術用語
- JavaScript（`×` Javascript）
- TypeScript（`×` Typescript）
- GitHub（`×` Github）
- macOS（`×` MacOS）

## SNS投稿ガイドライン

### X（Twitter）
- 1投稿1メッセージ
- ハッシュタグは2-3個まで
- 画像/動画があればエンゲージメント2-3倍
- 投稿時間: 平日 8:00, 12:00, 18:00 が最適

### LinkedIn
- 長文OK（1300字まで）
- ビジネス文脈の専門的な内容
- パーソナルな経験+学びの構成が効果的
- 投稿時間: 火-木 8:00-10:00 が最適

### ブログ
- タイトルにキーワードを含める
- meta description は120字以内
- 見出し（H2, H3）を適切に使用
- 内部リンク・外部リンクを含める

## プレスリリース配信ルール

- 配信前に代表の承認を得ること
- 数字・固有名詞のダブルチェック必須
- 配信曜日: 火-木（月金は埋もれやすい）
- 配信時間: 10:00-11:00
