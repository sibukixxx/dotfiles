# 課題・施策カタログ

石油情報システム更改報告書から抽出した、行政統計調査システムに共通する課題と施策の体系的カタログ。

---

## 1. 調査票収集チャネルの統合

### 課題
| No | 課題内容 | 影響を受ける利用者 |
|----|---------|-----------------|
| C-1 | 報告入力プログラムのインストールが社内規定で禁止される企業がある | 報告事業者 |
| C-2 | チャネルごとにデータ形式・入力I/Fが異なり処理が煩雑 | 運用事業者 |
| C-3 | e-Govからのダウンロードに時間制約があり取込にタイムラグが発生 | 運用事業者 |
| C-4 | 業界団体システムの仕様変更で取込不可になるリスク | 運用事業者 |
| C-5 | FAXは誤送信リスクがあり統計調査に不適切 | 全体 |

### 施策
| No | 施策内容 | 難易度 | 重要度 | 時期 |
|----|---------|--------|--------|------|
| M-1 | Web入力フォームの新規開発とチャネル統合 | 中 | 高 | 短期 |
| M-2 | 報告入力プログラムの段階的廃止 | 中 | 高 | 短期 |
| M-3 | 郵送チャネルの当面維持（移行期間配慮） | 低 | 高 | 短期 |
| M-4 | FAX・Excel添付メールは緊急措置のみ許可 | 低 | 中 | 短期 |

## 2. 業務効率化（調査票提出業務）

### 課題
| No | 課題内容 |
|----|---------|
| E-1 | 調査票形式の変更に即座に対応できない |
| E-2 | 企業名・企業IDの差し込み印刷を手動でVBA実施 |
| E-3 | 未記入の状態で調査票が提出されてしまう |

### 施策
| No | 施策内容 | 実装方針 |
|----|---------|---------|
| M-5 | 調査票フォーマット編集機能 | 管理画面からノーコードで項目追加・削除可能にする |
| M-6 | 企業情報の自動差込 | ログインアカウントに紐づく企業情報を自動表示 |
| M-7 | バリデーションチェック | 各ページに必須チェックを定義し、未記入を提出不可に |

## 3. 業務効率化（調査票受領・登録業務）

### 課題
| No | 課題内容 |
|----|---------|
| E-4 | 報告状況確認画面に企業名が表示されない |
| E-5 | 重複受領した調査票の差異確認が目視のみ |
| E-6 | エラーメッセージに箇所・原因の詳細がない |
| E-7 | 不備の修正に調査票の再取込が必要（直接修正不可） |
| E-8 | 修正履歴が残らず手書きでコピー管理 |
| E-9 | 一度取込エラーが出ると全調査票の取込が止まる |

### 施策
| No | 施策内容 | 実装方針 |
|----|---------|---------|
| M-8 | 報告状況画面に企業名・事業所名を表示 | データ参照画面の項目追加 |
| M-9 | 重複調査票の差異比較画面 | 差分ハイライト表示 |
| M-10 | エラー詳細表示（箇所・原因・修正案） | エラーハンドリング強化 |
| M-11 | 画面上での直接修正機能 | 項目単位のインライン編集 |
| M-12 | 変更履歴の自動記録・表示 | 監査ログ機能 |
| M-13 | エラー時の個別処理（他調査票は継続取込） | エラー分離処理 |
| M-14 | 石油事業者側での修正・取下げ機能 | Web画面から申請内容の修正・取下げ |

## 5. データ処理・運用効率化

### 課題
| No | 課題内容 |
|----|---------|
| E-16 | マスタデータが複数箇所に分散し二重登録 |
| E-17 | 個票ロック未実施で他処理が実行され異常停止 |
| E-18 | 年間補正で履歴データが保持されない |
| E-19 | タスクスケジューラの手動起動でオペミスリスク |
| E-20 | 行政側からシステムに直接アクセスできない |

### 施策
| No | 施策内容 | 実装方針 |
|----|---------|---------|
| M-21 | マスタデータ管理の一元化 | システム内包、API経由で同期 |
| M-22 | 処理依存関係の自動制御 | ワークフローエンジンで順序制御 |
| M-23 | 履歴データの保持・差分表示 | 変更前データをスナップショット保存 |
| M-24 | 画面ベースの処理実行 | タスクスケジューラ廃止、UI操作に |
| M-25 | ブラウザ経由の行政アクセス | クラウド化+ロールベースアクセス制御 |
