# 画面設計指針と操作性要件

次期統計調査システムの画面設計における指針と具体的な要件。

---

## 画面設計の3原則

1. **インストール不要** — ブラウザのみで全操作が完結すること
2. **ITリテラシー非依存** — 操作に迷わないわかりやすいUIであること
3. **旧システムの良さを継承** — 旧システムで好評だった操作感は維持すること

## 画面種別ごとの設計指針

### Type 1: データ編集画面

**設計原則**: 項目単位でのインライン編集を基本とする

```
要件:
- 各フィールドをクリックで直接編集モードに入れること
- 編集中の項目がハイライトされること
- バリデーションエラーは該当項目の直下にリアルタイム表示
- 保存前に変更箇所のプレビュー（差分表示）を提供
- Undo/Redo機能の実装
```

**React実装パターン**:
```typescript
interface EditableFieldProps {
  value: string | number;
  onSave: (newValue: string | number) => Promise<void>;
  validation?: (value: string | number) => string | null;
  fieldType: 'text' | 'number' | 'select';
}

const EditableField: React.FC<EditableFieldProps> = ({ value, onSave, validation, fieldType }) => {
  const [editing, setEditing] = useState(false);
  const [editValue, setEditValue] = useState(value);
  const [error, setError] = useState<string | null>(null);

  const handleSave = async () => {
    if (validation) {
      const err = validation(editValue);
      if (err) { setError(err); return; }
    }
    await onSave(editValue);
    setEditing(false);
  };

  // ... render logic
};
```

### Type 2: データ参照画面

**設計原則**: 表示項目に漏れがなく、データ出力が可能

```
要件:
- 企業名・企業ID・事業所名等の識別情報を必ず表示
- フィルタリング・ソート機能
- ページネーション（大量データ対応）
- CSV/Excelへのエクスポート機能
- 印刷用レイアウトの提供
```

**React実装パターン**:
```typescript
interface DataTableProps<T> {
  data: T[];
  columns: ColumnDef<T>[];
  exportable: boolean;
  filterable: boolean;
  sortable: boolean;
}

// エクスポート機能

  const csv = [headers, ...rows].join('\n');
  // BOM付きUTF-8でダウンロード（Excel日本語対応）
  const blob = new Blob(['\uFEFF' + csv], { type: 'text/csv;charset=utf-8;' });
  // ... download trigger
};
```

### Type 3: 機能実行画面

**設計原則**: 迷いなく操作でき、処理状況が可視化される

```
要件:
- 実行ボタンの明確な配置
- 実行前の確認ダイアログ
- 処理中のプログレスバー表示
- 処理完了後の結果サマリ表示
- エラー発生時の詳細表示とリトライ機能
- 処理依存関係の自動制御（前提条件未完了時はボタン非活性）
```

## 調査票フォーマット編集画面の設計

管理者がノーコードで調査票のWeb入力フォームを編集できる画面。

```
機能要件:
1. 調査項目の追加（項目名、データ型、必須/任意、選択肢設定）
2. 調査項目の削除（既存データへの影響確認付き）
3. 調査項目の並び替え（ドラッグ&ドロップ）
4. ページ構成の変更
5. バリデーションルールの設定
6. プレビュー機能（編集結果を事前確認）
7. バージョン管理（過去のフォーマットを参照可能）
```

---

## アクセシビリティ要件

行政システムとして JIS X 8341-3:2016 適合レベルAAに準拠すること。

- キーボードのみでの操作が可能
- スクリーンリーダー対応（適切なaria属性）
- 色のみに依存しない情報伝達
- 十分なコントラスト比の確保
- フォーム要素への適切なラベル付け
