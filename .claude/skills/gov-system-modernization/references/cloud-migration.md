# クラウド移行チェックリスト

行政統計調査システムのオンプレミスからクラウドへの移行指針。

---

## クラウド・バイ・デフォルト原則

政府情報システムは、クラウドサービスの利用を第一候補として検討する。
以下の順序で検討すること：

1. **SaaS** — 既製品で要件を満たせるか
2. **PaaS** — アプリ開発のみでよいか
3. **IaaS** — インフラ構築が必要か
4. **オンプレミス** — クラウド不可の場合のみ

---

## 移行前チェックリスト

### セキュリティ要件
- [ ] ISMAP（政府情報システムのためのセキュリティ評価制度）登録済みサービスか
- [ ] データの所在地（国内リージョン指定）
- [ ] 暗号化要件（保存時・転送時）
- [ ] アクセス制御（IAM、ロールベース）
- [ ] 監査ログの取得・保管

### ネットワーク要件
- [ ] 行政側からのブラウザアクセス経路
- [ ] 運用事業者からのアクセス経路
- [ ] 報告事業者（約250社）からのインターネットアクセス
- [ ] e-Gov等外部システムとの連携経路（将来的な廃止を見据え最小限）

### 可用性要件
- [ ] 稼働率目標（統計調査の提出期限に合わせたSLA設定）
- [ ] バックアップ・リストア方針
- [ ] DR（災害復旧）方針
- [ ] 緊急時報告への対応（石油備蓄法に基づく即応性）

### 性能要件
- [ ] 同時アクセス数（提出期限直前のピーク想定）
- [ ] データ処理量（月次集計バッチの所要時間）
- [ ] レスポンスタイム目標

### 移行要件
- [ ] 既存データの移行方法（旧システム→新システム）
- [ ] 旧システムからのマクロ・帳票の移行
- [ ] 並行稼働期間の設定
- [ ] ロールバック手順

## 現行システムとの対比

| 項目 | 現行（As-Is） | 次期（Can-Be/To-Be） |
|------|--------------|---------------------|
| インフラ | 運用事業者建屋内オンプレ | クラウド（ISMAP登録） |
| アクセス | 運用事業者端末のみ | ブラウザ（全利用者） |
| データ授受 | メール・FAX・手交 | Web経由で直接取込 |
| バックアップ | 外付けHDD + ポータブルHDD | クラウドストレージ自動バックアップ |
| 帳票作成 | 旧システムマクロ | システム内帳票エンジン |
| 処理実行 | タスクスケジューラ手動起動 | Web画面からの操作実行 |
| 監視 | なし（障害時に状況把握不能） | クラウド監視サービス + アラート |
