# mtg-prep: 商談準備自動化スキル

商談前の情報収集・整理・提案骨子作成を自動化する。

## 前提条件

### 必須
- qmd がインストール済み（Obsidian ナレッジベース検索）
- WebSearch ツールが使用可能

### オプション（MCP連携で強化）
- Gmail MCP → 顧客とのメール履歴を自動取得
- Slack MCP → 社内の顧客関連やり取りを取得
- Google Calendar MCP → 過去のMTG情報を取得
- Circleback MCP → 過去の議事録・AI要約を取得
- Notion MCP → 案件管理ステータスを取得
- Google Drive MCP → 関連資料（提案書・契約書）を検索

## 使い方

```
/mtg-prep [会社名 or 担当者名]
```

引数が省略された場合、対話的にヒアリングする。

## Workflow

### Phase 1: ヒアリング（5秒）

ユーザーから以下を確認:
1. **相手先**: 会社名・担当者名
2. **目的**: 商談の種類（初回/フォロー/提案/クロージング）
3. **補足**: 特に確認したいこと、気になっていること（任意）

### Phase 2: 並列調査（3エージェント同時実行）

**重要**: 3つのエージェントを Agent ツールで **並列** に起動する。

#### Agent 1: 顧客コンテキスト調査
```
目的: 過去のやり取り・関係性の把握

実行内容:
1. qmd query "[会社名] [担当者名]" で Obsidian ノートを検索
   - 過去のMTGメモ、商談記録、提案内容
2. （MCP利用可能な場合）
   - Gmail: 直近のメールやり取り
   - Slack: 顧客関連の社内チャット
   - Circleback: 過去MTGの議事録・AI要約
   - Notion: 案件ステータス・パイプライン情報
3. 発見した情報を構造化してまとめる

出力:
- 過去のやり取りサマリ
- 前回MTGの宿題・未解決事項
- 相手の関心事・課題
```

#### Agent 2: 企業・業界Web調査
```
目的: 最新の企業情報と業界動向の把握

実行内容:
1. WebSearch で企業の最新ニュース（直近3ヶ月）
2. WebSearch で業界動向・競合情報
3. 企業Webサイトから事業概要・最新IR情報を取得（WebFetch）
4. （MCP利用可能な場合）
   - Google Drive: 関連する提案書・契約書を検索

出力:
- 企業概要（事業内容・規模・最近の動き）
- 業界トレンド（3つ以内に絞る）
- 競合他社の動向
- 商談に使えそうなネタ・話題
```

#### Agent 3: 提案ドラフト作成
```
目的: 商談のアジェンダと提案骨子の作成

実行内容:
1. Agent 1, 2 の結果を待つ（依存あり）
   ※ 実際には Agent 1, 2 と並列開始するが、
   qmd のローカルノートだけで仮説ベースの骨子を先に作り始める
2. 過去の提案内容・顧客の課題から仮説を立てる
3. アジェンダ案を作成

出力:
- 推奨アジェンダ（タイムライン付き）
- 仮説ベースの提案ポイント（3つ以内）
- 確認すべき質問リスト
- リスク・注意点
```

### Phase 3: 統合・ブリーフィング資料生成

3つのエージェントの結果を統合し、以下の形式で出力する。

## Save Location

`notes/docs/records/mtg-prep/[会社名]-YYYY-MM-DD.md`

ディレクトリが存在しない場合は作成する。

## MCP 連携なしでの動作

MCP サーバーが未設定でも、以下だけで実用的な資料を生成できる:

1. **qmd** → Obsidian の過去ノートから顧客情報を検索
2. **WebSearch** → 企業・業界の最新情報を取得
3. **ユーザー入力** → 補足情報を対話的にヒアリング

MCP を追加接続するたびに、資料の情報密度が上がる設計。

## Quality Checklist

- [ ] 過去のやり取りが反映されているか
- [ ] 企業・業界情報が最新か（3ヶ月以内）
- [ ] アジェンダが商談の目的に沿っているか
- [ ] 提案ポイントが具体的で根拠があるか
- [ ] 確認質問が漏れなくリストアップされているか
