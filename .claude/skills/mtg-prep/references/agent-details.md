# mtg-prep: エージェント詳細仕様

## Agent 1: 顧客コンテキスト調査

```
目的: 過去のやり取り・関係性の把握

実行内容:
1. qmd query "[会社名] [担当者名]" で Obsidian ノートを検索
   - 過去のMTGメモ、商談記録、提案内容
2. （MCP利用可能な場合）
   - Gmail: 直近のメールやり取り
   - Slack: 顧客関連の社内チャット
   - Circleback: 過去MTGの議事録・AI要約
   - Notion: 案件ステータス・パイプライン情報
3. 発見した情報を構造化してまとめる

出力:
- 過去のやり取りサマリ
- 前回MTGの宿題・未解決事項
- 相手の関心事・課題
```

## Agent 2: 企業・業界Web調査

```
目的: 最新の企業情報と業界動向の把握

実行内容:
1. WebSearch で企業の最新ニュース（直近3ヶ月）
2. WebSearch で業界動向・競合情報
3. 企業Webサイトから事業概要・最新IR情報を取得（WebFetch）
4. （MCP利用可能な場合）
   - Google Drive: 関連する提案書・契約書を検索

出力:
- 企業概要（事業内容・規模・最近の動き）
- 業界トレンド（3つ以内に絞る）
- 競合他社の動向
- 商談に使えそうなネタ・話題
```

## Agent 3: 提案ドラフト作成

```
目的: 商談のアジェンダと提案骨子の作成

実行内容:
1. Agent 1, 2 の結果を待つ（依存あり）
   ※ 実際には Agent 1, 2 と並列開始するが、
   qmd のローカルノートだけで仮説ベースの骨子を先に作り始める
2. 過去の提案内容・顧客の課題から仮説を立てる
3. アジェンダ案を作成

出力:
- 推奨アジェンダ（タイムライン付き）
- 仮説ベースの提案ポイント（3つ以内）
- 確認すべき質問リスト
- リスク・注意点
```
