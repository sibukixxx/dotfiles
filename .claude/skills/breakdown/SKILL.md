# Breakdown - タスク分解支援

大きなタスクを「今すぐ着手できる」粒度（25分以内）まで分解し、次の一歩を明確にする。

## 基本原則

### なぜ25分か？

- **ポモドーロ・テクニック**: 集中力が持続する単位
- **心理的ハードル**: 「25分だけ」なら着手しやすい
- **進捗の可視化**: 短いサイクルで達成感を得られる

### 分解の基準

```
良いタスク粒度:
✓ 25分以内で完了できる
✓ 完了状態が明確
✓ 次のアクションが具体的
✓ 依存関係がない（単独で着手可能）

悪いタスク粒度:
✗ 「〜を検討する」（終わりが不明確）
✗ 「〜を実装する」（大きすぎる）
✗ 「〜を調べる」（アウトプットが不明確）
```

## プロセス

### Step 1: 現状把握

**確認する情報:**

1. **元のタスク**: 何を達成したいか
2. **現在の進捗**: どこまで終わっているか
3. **現在時刻**: 残り作業時間はどれくらいか
4. **ブロッカー**: 何が手を止めているか

```markdown
## 現状整理

- 元タスク: [タスク名]
- 進捗: [0-100%]
- 残り時間: [今日の残り作業時間]
- 困っていること: [ブロッカー]
```

### Step 2: タスクの構造分析

**分解のアプローチ:**

| アプローチ | 説明 | 適用場面 |
|------------|------|----------|
| **工程分解** | 手順に沿って分ける | 実装タスク |
| **成果物分解** | アウトプットで分ける | 作成タスク |
| **範囲分解** | スコープを絞る | 調査タスク |
| **時間分解** | 時間で区切る | 反復タスク |

### Step 3: 25分タスクに分解

**分解テンプレート:**

```markdown
## 分解結果

### 今すぐできるタスク（25分以内）
1. [ ] [具体的なタスク] (約Xmin)
2. [ ] [具体的なタスク] (約Xmin)
3. [ ] [具体的なタスク] (約Xmin)

### 次のセッションでやるタスク
4. [ ] [タスク]
5. [ ] [タスク]

### ブロック解消が必要なタスク
- [ ] [タスク] ← 必要: [何が必要か]
```

### Step 4: 最初の一歩を特定

**次のアクション決定:**

```markdown
## 次のアクション

**今から25分でやること:**
[最も着手しやすいタスク]

**完了の定義:**
[何ができたら完了か]

**始め方:**
[具体的な最初のアクション]
```

## 分解パターン集

### パターン1: 実装タスク

```
元: 「ログイン機能を実装する」

分解:
1. [ ] ログインフォームのHTMLを書く (15min)
2. [ ] バリデーションロジックを書く (20min)
3. [ ] APIエンドポイントを作成する (25min)
4. [ ] フロントとAPIを繋ぐ (20min)
5. [ ] エラーハンドリングを追加 (15min)
6. [ ] 手動テストで動作確認 (10min)
```

### パターン2: 調査タスク

```
元: 「最適なライブラリを調べる」

分解:
1. [ ] 候補を3つに絞る (15min)
2. [ ] 各候補のGitHub Star/更新頻度を確認 (10min)
3. [ ] 1つ目のライブラリでHello World (25min)
4. [ ] 比較表を作成する (15min)
5. [ ] 推奨を1つ決めて理由を書く (10min)
```

### パターン3: 文書作成タスク

```
元: 「設計ドキュメントを書く」

分解:
1. [ ] アウトラインを箇条書きで書く (15min)
2. [ ] 背景セクションを書く (20min)
3. [ ] 技術選定セクションを書く (25min)
4. [ ] 図を1つ作成する (20min)
5. [ ] レビュー依頼を出す (5min)
```

### パターン4: バグ修正タスク

```
元: 「〇〇のバグを直す」

分解:
1. [ ] バグを再現する手順を確認 (10min)
2. [ ] 原因箇所を特定する (25min)
3. [ ] 修正方針を決める (10min)
4. [ ] 修正を実装する (20min)
5. [ ] 再現手順で修正を確認 (10min)
```

## 時間帯別アドバイス

| 残り時間 | 推奨アプローチ |
|----------|----------------|
| **2時間以上** | 25分タスク x 3-4個を計画 |
| **1-2時間** | 最重要タスク1つに集中 |
| **30分-1時間** | 25分タスク1つ + 整理 |
| **30分未満** | 次回の準備（メモ残し）のみ |

## ブロッカー別対処法

### 「何から始めればいいかわからない」

→ **最小のアウトプットを定義する**
- ファイルを1つ作る
- 関数シグネチャだけ書く
- 見出しだけ書く

### 「全体像が見えない」

→ **わかっている部分から始める**
- 確実にやることをリストアップ
- 不明点は「TODO: 要確認」で進む

### 「完璧にしたい」

→ **v0.1を目指す**
- まず動くものを作る
- 改善は次のタスクとして分離

### 「割り込みが多い」

→ **防御時間を確保**
- 「25分だけ集中」と宣言
- 通知をオフにする
