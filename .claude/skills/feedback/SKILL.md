# Feedback - 行動分析学ベース企画レビュー

ユーザーの行動変容を目的とした企画を、行動分析学のフレームワークでレビューし、改善案と検証計画を提供する。

## モード選択

```
/feedback           → general モード（デフォルト）
/feedback --pr      → PR モード
/feedback --general → general モード
```

### モード判定基準

| モード | 対象 | 主眼 |
|--------|------|------|
| **general** | 機能、施策、キャンペーン、UX改善 | ユーザー行動の変容 |
| **pr** | プレスリリース、広報、マーケコンテンツ | 情報接触後の行動 |

## 行動分析学フレームワーク

### ABC分析（基本フレーム）

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ Antecedent  │ →  │  Behavior   │ →  │ Consequence │
│   先行条件   │    │    行動     │    │    結果     │
└─────────────┘    └─────────────┘    └─────────────┘
```

- **Antecedent（先行条件）**: 行動を引き起こすきっかけ・環境・刺激
- **Behavior（行動）**: ユーザーに取ってほしい具体的な行動
- **Consequence（結果）**: 行動後に得られる報酬・フィードバック

### 強化の4象限

| | 刺激を加える | 刺激を除去する |
|---|---|---|
| **行動を増やす** | 正の強化（報酬を与える） | 負の強化（不快を除去） |
| **行動を減らす** | 正の罰（不快を与える） | 負の罰（報酬を除去） |

## General モード - レビュープロセス

### Step 1: 企画の行動定義

**確認すべき項目：**

1. **ターゲット行動は何か？**
   - 具体的で観察可能か
   - 測定可能か（頻度・持続時間・強度）

2. **誰の行動か？**
   - ターゲットユーザーは明確か
   - セグメントは適切か

3. **現状の行動は？**
   - ベースラインは把握しているか
   - なぜ現在その行動をしない/しているのか

### Step 2: ABC分析

**分析チェックリスト：**

```markdown
## ABC分析シート

### Antecedent（先行条件）
- [ ] 行動のトリガーは明確か
- [ ] ユーザーがトリガーに気づく仕組みはあるか
- [ ] トリガーのタイミングは適切か
- [ ] 障壁は特定されているか

### Behavior（行動）
- [ ] 求める行動は1つに絞られているか
- [ ] 行動のハードルは適切か
- [ ] 行動の手順は明確か

### Consequence（結果）
- [ ] 行動後のフィードバックはあるか
- [ ] 報酬は即時か
- [ ] 報酬は行動に見合っているか
- [ ] 繰り返しを促す仕組みはあるか
```

### Step 3: 強化設計の評価

**評価観点：**

| 観点 | 質問 | 良い例 | 悪い例 |
|------|------|--------|--------|
| 即時性 | 報酬はすぐ得られるか | 完了時のアニメーション | 月末のポイント付与 |
| 確実性 | 報酬は必ず得られるか | 毎回のフィードバック | 抽選でプレゼント |
| 価値 | ユーザーにとって価値があるか | 時間節約 | 運営都合のバッジ |
| 可視性 | 進捗が見えるか | プログレスバー | 完了まで何もなし |

### Step 4: 改善提案

以下のフォーマットで提案：

```markdown
## 改善提案

### 課題
[ABC分析で特定した課題]

### 改善案
[具体的な改善内容]

### 期待効果
[行動指標でどう変わるか]

### 実装難易度
[低/中/高]
```

### Step 5: 検証計画

```markdown
## 検証計画

### 仮説
「[先行条件]の状況で[ターゲット行動]をすると[結果]が得られるため、行動頻度が[X%]向上する」

### 測定指標
- 主指標: [行動の頻度/回数/率]
- 副指標: [関連する行動指標]
- ガードレール: [悪化を検知する指標]

### 検証方法
- [ ] A/Bテスト
- [ ] Before/After比較
- [ ] コホート分析

### サンプルサイズ・期間
[必要なサンプルと期間の見積もり]
```

## PR モード - レビュープロセス

PRモードでは、情報接触後の行動変容に焦点を当てる。

### Step 1: コミュニケーション目標の行動定義

**確認すべき項目：**

1. **期待する行動は何か？**
   - 記事を読む → その後何をしてほしい？
   - サービス認知 → 次のアクションは？

2. **行動の連鎖設計**
   ```
   認知 → 興味 → 検索 → 訪問 → 試用 → 継続
   ```

### Step 2: メッセージのABC分析

| 要素 | 分析観点 |
|------|----------|
| **A: 先行条件** | どんな状況で読まれるか、読者の事前知識は |
| **B: 期待行動** | 読後にとってほしい具体的アクション |
| **C: 提示する結果** | 行動のメリット、しないデメリット |

### Step 3: 行動喚起の評価

**チェックリスト：**

- [ ] CTA（Call to Action）は明確か
- [ ] 行動の障壁は低いか
- [ ] 行動のメリットは具体的か
- [ ] 社会的証明はあるか
- [ ] 緊急性・希少性は適切か（煽りすぎない）

### Step 4: 改善提案・検証計画

Generalモードと同様のフォーマットで提案。PRモードでは以下も追加：

```markdown
## メディア・チャネル設計

### ターゲットの情報接触行動
- どこで情報を得るか
- いつ情報を得るか
- 誰の情報を信頼するか

### チャネル選定根拠
[行動分析に基づく理由]
```
