# 強化スケジュール

## 概要

強化スケジュールとは、行動に対していつ・どのくらいの頻度で強化を与えるかのパターン。スケジュールによって行動の獲得・維持・消去パターンが異なる。

## 連続強化 vs 間欠強化

| 種類 | 説明 | 特徴 |
|------|------|------|
| 連続強化 | 毎回強化 | 獲得が早い、消去も早い |
| 間欠強化 | 時々強化 | 獲得は遅い、消去に強い |

**設計指針**:
- 新規行動の獲得 → 連続強化から開始
- 行動の維持 → 間欠強化に移行

## 4つの基本スケジュール

### 1. 固定比率（FR: Fixed Ratio）

一定回数の行動ごとに強化。

```
行動 → 行動 → 行動 → 強化
（例: 10回購入で1回無料）
```

| 特徴 | 企画への応用 |
|------|-------------|
| 高い反応率 | スタンプカード（10個で特典） |
| 休止期間あり | ポイント還元（1000円で10pt） |
| 予測可能 | レベルアップ（経験値固定） |

### 2. 変動比率（VR: Variable Ratio）

平均N回の行動ごとに強化（回数は変動）。

```
行動 → 強化 → 行動 → 行動 → 行動 → 強化
（例: ガチャ、スロット）
```

| 特徴 | 企画への応用 |
|------|-------------|
| 非常に高い反応率 | ガチャ・抽選 |
| 消去抵抗が強い | ランダムボーナス |
| 依存性あり | いいね・リツイート |

**注意**: VRは依存性を生みやすい。倫理的配慮が必要。

### 3. 固定間隔（FI: Fixed Interval）

一定時間経過後の最初の行動を強化。

```
時間経過... → 行動 → 強化
（例: 日次ログインボーナス）
```

| 特徴 | 企画への応用 |
|------|-------------|
| 時間後に行動増加 | ログインボーナス |
| スキャロップ型 | タイムセール |
| 待機時間あり | スタミナ回復 |

### 4. 変動間隔（VI: Variable Interval）

平均N時間経過後の最初の行動を強化（時間は変動）。

```
時間経過(変動)... → 行動 → 強化
（例: メールのチェック）
```

| 特徴 | 企画への応用 |
|------|-------------|
| 安定した反応率 | 通知チェック |
| 消去抵抗が強い | フィード更新 |
| チェック行動を誘発 | 在庫チェック |

## 複合スケジュール

### 並立スケジュール（Concurrent）

複数のスケジュールが同時に存在。

```
行動A → スケジュール1 → 強化
行動B → スケジュール2 → 強化
```

**応用例**: 複数の行動オプションを提示し、ユーザーに選択させる

### 連鎖スケジュール（Chain）

複数のスケジュールが順番に連結。

```
スケジュール1 完了 → スケジュール2 開始 → ... → 最終強化
```

**応用例**: オンボーディング、チュートリアル

## 企画設計の指針

### 行動フェーズ別の推奨

| フェーズ | 推奨スケジュール | 理由 |
|----------|-----------------|------|
| 獲得期 | 連続強化 (CRF) | 行動と結果の対応を学習 |
| 強化期 | 固定比率 (FR) | 予測可能で達成感 |
| 維持期 | 変動比率 (VR) | 消去に強い |
| 習慣化 | 変動間隔 (VI) | 安定した行動 |

### スケジュール移行のタイミング

1. **CRF → FR**: 行動が安定して出現するようになったら
2. **FR → VR**: 固定パターンに慣れたら
3. **VR → VI**: 時間ベースの習慣に移行したい場合

## 注意事項

### 倫理的配慮

- VRスケジュールは依存性を生みやすい（ギャンブル依存症の原理）
- 特にゲーム・SNSでは過度な使用を促さない設計を
- 「休憩を促す」「使用時間の可視化」などの対策

### 消去バースト

強化を停止すると、一時的に行動が増加する現象。

```
強化停止 → 行動急増（消去バースト）→ 行動減少
```

**設計への示唆**: 仕様変更時には消去バーストへの対策を検討
