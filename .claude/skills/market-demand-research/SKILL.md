# 案件・発注者の悩みリサーチ（毎朝10分）

目的：市場を読むとは、**お金を払う人の悩みを読む**こと。発注文にはリアルな悩みがそのまま書かれている。これを毎日1個抜き出す。

## 手順

### 1. 今日見るサイトを1つ決めてもらう
ローテーション候補：

- クラウドワークス / ランサーズ / ココナラ
- Xの募集投稿（「募集」「外注」「業務委託」で検索）
- Wantedly / YOUTRUST / 複業クラウド / Workship
- レバテック系 / ITプロパートナーズ系 / Indeed

ユーザーのジャンルが分かっていれば、検索キーワード（例：「SNS運用 代行」「LP 改善」）まで具体的に提案する。5分だけ見てもらう。

### 2. 発注文を1件（余裕があれば2〜3件）貼ってもらう
貼られた発注文を、このテンプレでClaudeが構造化する：

```
悩み：（例）SNSを毎日更新できない
背景：（例）社内に担当者がいない
欲しい結果：（例）投稿を継続して認知を増やしたい
払う理由：（例）自社でやる時間がないから
提案できる商品：（例）SNS投稿作成代行 / 投稿テンプレート / AI運用フロー構築
```

「提案できる商品」はClaudeが2〜3案出し、**スキルではなく悩みの解決策**として言い換える。例：「ライティングします」ではなく「売れるLPを作ります」、「AI使えます」ではなく「投稿作成を3時間→30分にします」。

### 3. 案件を評価する
貼られた案件について、以下の観点で1〜2行コメントする：

- 単価帯はどこか／継続案件になりそうか／初心者でも取れるか
- 高単価化・パッケージ化できそうか／AIで効率化できる余地はどこか

### 4. 「お金に近いか」を分類する
悩みを3分類のどれかにタグ付けする：

- **お金に近い市場**（集客・営業・採用・広告・LP改善・SNS運用・自動化など）→ 企業が払いやすい
- **面倒の代行市場**（投稿作成・資料作成・リサーチ・事務・請求書など）→ 「やりたくないけど必要」は売れやすい
- **AI効率化市場**（投稿量産・営業メール・FAQ・議事録・レポートなど）→ 個人でも戦いやすい

### 5. ログを出力する
`/mnt/user-data/outputs/悩みリサーチ_YYYY-MM-DD.md` に保存して `present_files` で渡す。末尾に累計目安（週10個 → 月40個 → 3ヶ月120個）への進捗を一言添えて、蓄積を実感させる。

## 注意

- 1日1個でいい。質より継続。
- 発注文が貼られず「今日は見る時間なかった」と言われたら、ユーザーのジャンルでよくある発注文の型を1つ提示し、「明日これに近いものを探してみて」と宿題化する（Claudeの創作を実データとして記録しない。創作した場合は仮想例と明記する）。
