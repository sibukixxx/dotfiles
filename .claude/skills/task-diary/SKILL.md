# task-diary: タスク日記・セッション学習記録

セッションで得た学びを構造化し、複利的に知識を蓄積する。

> Anthropic社内パターン: セッションごとの学びを日記→observations→CLAUDE.mdへ昇華

## 使い方

- `/task-diary` — 現セッションの学びを記録
- `/task-diary review` — 蓄積された日記を分析し、observationsを生成
- `/task-diary integrate` — observationsをCLAUDE.mdに統合提案

## Workflow

### Step 1: セッション振り返り

現セッションで行った作業を振り返り、以下を構造化する:

1. **何をやったか**: タスクの概要
2. **何がうまくいったか**: 成功パターン・効率的だったアプローチ
3. **何がうまくいかなかったか**: 失敗・手戻り・非効率だった点
4. **なぜうまくいった/いかなかったか**: 原因分析
5. **次回への教訓**: 具体的なアクション

### Step 2: 日記エントリの作成

以下のフォーマットで `.context/task-diary/` に保存:

```markdown
---
date: YYYY-MM-DD
session_id: [セッションIDの先頭8文字]
duration: [概算作業時間]
tags:
  - [関連タグ]
---

## やったこと
- [タスク1の概要]
- [タスク2の概要]

## うまくいったこと
- [成功パターン]: [なぜうまくいったか]

## うまくいかなかったこと
- [失敗/手戻り]: [原因] → [次回の対策]

## 発見・気づき
- [技術的な発見や新しい知見]

## 次回への教訓
- [ ] [具体的なアクションアイテム]
```

### Step 3: Observations の生成（`review` モード）

蓄積された日記（3件以上）を分析し、パターンを抽出:

```markdown
## Observations（YYYY-MM-DD時点）

### 繰り返し成功しているパターン
1. [パターン名]: [説明]

### 繰り返し失敗しているパターン
1. [パターン名]: [説明] → [推奨対策]

### 新たに確立されたベストプラクティス
1. [プラクティス名]: [説明]
```

### Step 4: CLAUDE.md 統合提案（`integrate` モード）

Observationsの中から、プロジェクト全体に適用すべきルールを特定し、
CLAUDE.mdへの追記案を提示する。ユーザー確認後に反映。

## 保存先

```
.context/task-diary/
├── YYYY-MM-DD-[session_id].md   # 個別日記
├── observations.md               # 蓄積された観察結果
└── archive/                      # 統合済みの古い日記
```

## 蓄積モデル

```
セッション1: 日記 → 学び3つ
セッション2: 日記 → 学び2つ + 前回の教訓を活用
セッション3: 日記3件 → review → observations生成
セッションN: observations → integrate → CLAUDE.md更新
→ 以降、全セッションが賢くなる（複利的蓄積）
```

## Troubleshooting

### 日記が形式的になりがちな場合
「なぜ」を必ず書く。「テストを先に書いた→うまくいった」ではなく「テストを先に書いた→仕様の曖昧さに早期に気づけた→手戻りが減った」。

### 蓄積が多すぎて見返せない場合
`review` モードで定期的にobservationsに圧縮。統合済みの日記はarchiveに移動。

### CLAUDE.mdが肥大化しそうな場合
本当にプロジェクト全体に適用すべきルールだけをintegrate。個人的な学びはobservationsに留める。
